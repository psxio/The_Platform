Multi-Portal Consolidation Guide
Overview
Consolidate multiple SIGNALS portal instances (Tenge, Fireside, Titanium, RYFT) into the main backend at wl.theagency.vc, allowing all portals to share one codebase while maintaining separate branding, data, and admin access.

Current Setup
Portal	Domain	Replit Instance	Status
The Agency (4444)	wl.theagency.vc	THIS INSTANCE	Live
Fireside	w.firesidebase.com	Separate Replit	Live
Titanium	wl.titanium.finance	Separate Replit	Live
RYFT	wl.entertheryft.com	Separate Replit	Live
Tenge	tenge--psxonbase.replit.app	Separate Replit	Test
Target Setup
All domains point to this single backend. Domain detection loads the correct theme/whitelist automatically.

How It Works
User visits wl.titanium.finance
Backend detects hostname from request
Database lookup finds whitelist where domains array contains "titanium.finance"
Returns project config (theme, whitelist ID, branding)
Frontend loads Titanium theme, shows only Titanium data
Backend Changes Required
1. Database: Add Domains Column
ALTER TABLE whitelists ADD COLUMN domains TEXT[] DEFAULT '{}';

Each whitelist gets its domains:

Tenge: ["tenge--psxonbase.replit.app", "wl.tenge.xyz"]
Fireside: ["firesidebase.com", "w.firesidebase.com"]
2. New API Endpoint
GET /api/resolve-project?host=wl.titanium.finance

Returns:

{
  "whitelistId": 7,
  "slug": "titanium",
  "theme": "titanium",
  "name": "Titanium Finance"
}

3. Admin Project Scoping
New table admin_project_roles:

Links admin wallet â†’ whitelist IDs they can access
Superadmins see everything
Project admins only see their project
